# AEM [!DNL Forms]發行{#aem-forms-releases}

Adobe Experience Manager [!DNL Forms]是部署在AEM上的應用程式。 它將引人入勝的網路和行動體驗延伸至企業表單和檔案，讓組織能夠完成複雜的數位互動，同時改善使用者體驗並擴展業務觸及面。 AEM Quick start隨附的現成可用AEM [!DNL Forms]有一組有限的功能。 AEM [!DNL Forms]附加套件提供進階功能，例如與[!DNL Adobe Analytics]、[!DNL Adobe Sign]、檔案服務、工作流程引擎等整合。

>[!NOTE]
>
>AEM Forms會在排程的AEM Service Pack和Cumulative Fix Pack發行日期一週後發行附加套件。

下列清單包含所有今天之前發行的AEM [!DNL Forms]附加元件套件、對應的AEM版本（預先要求）、套件的下載連結，以及其他實用資訊：

## 要記住的事項{#things-to-remember}

* 上述AEM Forms附加元件套件適用於OSGi上的AEM [!DNL Forms]和JEE上的AEM [!DNL Forms]。
* 所有AEM [!DNL Forms]附加元件套件和快速修正都是累積的；也就是說，附加元件套件和快速修正包含對應版本之元件的前述所有附加元件和快速修正。 例如，如果您要安裝Quick Fix 2.4.50，它已包含AEM [!DNL Forms]附加套件2.4.26。

   >[!NOTE]
   >
   >累積附加套件和快速修正表示，前兩個版本的編號完全相同。

* 請務必安裝最新可用的Service Pack/Feature Pack附加元件。 強烈建議您使用它，因為它包括安全性、效能、穩定性，以及自基本版本全面推出以來所發佈的重要客戶修正和增強功能。
* 每當您安裝Service Pack或從舊版AEM升級至較新版本時，請務必安裝對應的AEM [!DNL Forms]附加套件。 例如，如果您從AEM 6.1 [!DNL Forms]附加套件2.2.4版升級至AEM 6.1 Service Pack 1，在安裝AEM 6.1 Service Pack 1後，請安裝AEM [!DNL Forms]附加套件2.4.50版。
* 不支援解除安裝AEM [!DNL Forms]附加元件套件。
